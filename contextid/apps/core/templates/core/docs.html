User Guide

Profile

Your identity is not a single entity. It is a collection of Profiles tailored to specific environments.

  • Dashboard: Visit your Dashboard to view, create, and manage your identities.
  • One Per Context: You can create exactly one profile for each available context (e.g., one Legal profile, one Social profile).
  • Privacy Control: You can set your profile visibility to Public, allowing anyone to view it. Otherwise only you and admins will have access.
  • Dynamic Attributes: Enhance your profile with custom fields like Gender, Date of Birth, or Contact Number.
Supported Attribute Types:
STRING INT BOOL DATE URL

System Workflow

  1. Account Setup: Register and Login to the platform.
  2. Profile Management: Navigate to the Dashboard to build your profiles. Select which ones are Public (visible to anyone) or Private (visible only to you and admins).
  3. Preview: View your profile to see how it will be presented to others in that specific context.
  4. Sharing: Click the save button under your profile to save and share your contextual identity.
  5. Identity Resolution: Other users or developers can find your public profiles using: /api/v1/identity/?user={username}&context={context}

Authentication & Authorization

All API requests must include a valid JSON Web Token (JWT). Tokens expire after a set period for security.

# Request Header
Authorization: Bearer YOUR_ACCESS_TOKEN

Obtain your token via POST /api/v1/accounts/login/.
